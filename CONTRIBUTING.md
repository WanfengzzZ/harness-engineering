# 贡献指南

欢迎贡献 Harness Engineering Skill！

## 如何贡献

### 1. 报告问题

发现 Bug 或有改进建议？请提交 Issue：

- Bug 报告：请描述复现步骤
- 功能建议：请说明使用场景
- 文档改进：请指出具体问题

### 2. 提交代码

#### 准备工作

1. Fork 本仓库
2. Clone 到本地
3. 创建特性分支

```bash
git checkout -b feature/amazing-feature
```

#### 开发规范

1. **代码风格**
   - Python 脚本遵循 PEP 8
   - 添加必要的注释
   - 编写单元测试

2. **文档规范**
   - 使用简体中文
   - 保持格式一致
   - 添加必要的示例

3. **提交信息**
   - 使用现在时态
   - 以动词开头
   - 说明为什么改

#### 提交流程

```bash
# 提交变更
git commit -m "Add amazing feature"

# 推送到分支
git push origin feature/amazing-feature

# 创建 Pull Request
```

### 3. 审查流程

1. 项目维护者会审查 PR
2. 可能需要根据反馈修改
3. 审查通过后合并

## 贡献类型

### 代码改进

- 修复 Bug
- 新增功能
- 性能优化
- 重构代码

### 文档改进

- 修正错误
- 补充内容
- 改进示例
- 翻译文档

### 社区建设

- 分享使用案例
- 回答社区问题
- 推广最佳实践

## 行为准则

### 我们的承诺

为了营造开放友好的环境，我们承诺：

- 尊重不同观点和经验
- 接受建设性批评
- 关注对社区最有利的事
- 对其他社区成员表示同理心

### 不可接受的行为

- 使用性别化的语言或图像
- 人身攻击或侮辱性评论
- 公开或私下骚扰
- 未经许可发布他人隐私信息
- 其他不道德或不专业的行为

## 许可

## 问题？

如有任何问题，请联系项目维护者或提交 Issue。
